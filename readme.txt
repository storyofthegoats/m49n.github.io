This repository uses a predictable pattern for journal entries.
This workflow must be followed for every new entry.

1. User creates raw journal text

Matt writes a raw journal entry in any format.

2. AI generates interpreted version

Kev rewrites it in a â€œscaled-down interpretiveâ€ style.

3. AI generates the full entry file

Kev produces a complete HTML file with:

both sections formatted cleanly

same fonts and layout used across all entries

4. AI generates the index snippet

Kev also creates a single HTML block for journal.html linking to the new entry.

5. User commits the changes

Matt:

Adds the new HTML file under /entries/

Pastes the index snippet into journal.html

Commits and pushes to GitHub

GitHub Pages deploys automatically

This workflow ensures consistency and keeps the journal system scalable.
